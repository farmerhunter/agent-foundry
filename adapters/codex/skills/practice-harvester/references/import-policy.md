# External Import Policy

Canonical source: `workflows/import-external-skills.md`

External skills are reviewed inputs, not trusted source-of-truth content.

When borrowing from the internet:

1. Capture provenance.
2. Check license.
3. Review scripts and security risks.
4. Extract reusable candidates.
5. Deduplicate against canonical practices.
6. Present extracted candidates for human review.
7. After approval, canonicalize, promote to active when applicable, update the index, and publish relevant adapters.

Never execute external scripts or install dependencies without explicit approval.
