# Harvest Practices

Run the canonical practice harvesting workflow.

Steps:

1. Read `workflows/harvest-practices.md`.
2. Reconstruct the session.
3. Run the current capability check.
4. Route artifacts before abstracting practices.
5. Apply the generalization gate.
6. Extract reusable candidates.
7. Search `indexes/practice_index.yaml`.
8. Decide discard, merge, create, supersede, or defer.
9. Update canonical files first.
10. Ask before promoting to `active`.
11. After approval, publish relevant adapters automatically.
12. Report candidates, rejected-as-practice items, decisions, changed files, and review needs.
